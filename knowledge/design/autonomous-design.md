# Initial key points

- Designing for the invisible
- Explanaitions & interpretability — everywhere
- Fast Inference is contrary to trust, can be taken into advantage
- Visibility of actions, as if it were logs — but depends on the users’ trust so this might go away generally or as time passes within a system (onboarding-based)
- Traces and logs
- Intent autonomy, scheduled autonomy and background autonomy
- Changes of autonomy (e.g. ‘move to background workers’)
- Complexity for engineering; async states with different timelines
